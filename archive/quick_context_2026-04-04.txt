  ⚠️ GitHub Push — ปัญหาไฟล์ใหญ่เกิน 100MB
================================================================================

ปัญหา:
  GitHub ปฏิเสธ push ถ้ามีไฟล์เกิน 100MB ใน commit history
  (ถึงแม้จะลบไฟล์แล้ว ถ้ายังอยู่ใน commit เก่าก็ push ไม่ได้)

วิธีแก้ที่ใช้:
  push_to_github.bat มี Step 4.5 ที่ทำ git reset --soft origin/main
  ก่อน commit ใหม่ — เป็นการ squash unpushed commits ทิ้ง history เก่า
  + .gitignore มี: *.sqlite, *.db, *.cache.json, *.zip, *.rar, *.7z, *.bak

ถ้าเกิดอีกในอนาคต (ไฟล์ใหญ่หลุดเข้า git):
  วิธี A — ถ้ายัง push ไม่สำเร็จ (commits อยู่ local):
    git reset --soft origin/main
    git rm --cached "path/to/largefile"
    เพิ่ม pattern ใน .gitignore
    git add -A && git commit -m "Remove large file"
    git push

  วิธี B — ถ้า push สำเร็จแล้ว (ไฟล์อยู่ใน remote history):
    ใช้ BFG Repo-Cleaner (https://rtyley.github.io/bfg-repo-cleaner/)
    java -jar bfg.jar --strip-blobs-bigger-than 100M
    git reflog expire --expire=now --all
    git gc --prune=now --aggressive
    git push --force

  วิธี C — ถ้าจะเริ่มใหม่ (ง่ายสุด):
    ลบ .git folder
    git init && git remote add origin <url>
    git add -A && git commit -m "Fresh start"
    git push --force origin main
